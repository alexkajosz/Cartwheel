Duro Data Processing Addendum (DPA)

Last updated: February 8, 2026

Roles

Merchants are the data controllers. Duro acts as a data processor on behalf of merchants.

Data and purpose

  • Store and configuration data to run the app.
  • Operational logs for security and reliability.
  • Content inputs and outputs to generate and publish content.

Processing instructions

We process data only to provide the app features and support merchants.

Security measures

  • Encryption in transit (HTTPS).
  • Encryption at rest when enabled.
  • Access logging and least-privilege access.
  • Backups with encryption when required.

Subprocessors

We use infrastructure and AI providers to operate the service. A current list of subprocessors is available on request.

Data retention and deletion

We retain data only as long as needed. See the Data Retention Policy. Merchants can request deletion.

Contact

Contact us using the support email listed in the app listing.
